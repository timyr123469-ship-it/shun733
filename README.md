# Услуги по подбору цвета для профориентации

Сервис профориентации для детей, который предлагает направления и профессии
на основе выбранной цветовой палитры.

---

## Цель проекта

Создать MVP сервиса, в котором ребёнок:
1. Выбирает цветовую палитру
2. Получает рекомендации профессий
3. Видит результат в простой и понятной форме

---

## Команда и роли

- **Project Manager** - планирование, приоритизация, приёмка
- **Backend Developer** - API и бизнес-логика
- **Frontend Developer** - пользовательский интерфейс
- **QA Engineer** - тестирование и контроль качества

---

## Функциональные возможности

- Выбор цветовой палитры (UI)
- Интерпретация цветов и сопоставление с интересами
- Подбор 3-5 профессиональных направлений
- Отображение результата в детском UX
- Базовое логирование и конфигурация правил

---

## План проекта

- **Длительность:** 2 спринта по 1.5 недели
- **Общий лимит:** 220 часов
- **Подход:** MVP → улучшение UX и качества

---

## Планирование спринта и загрузка команды

**Sprint 1** - базовый функционал (110 часов)
*Фокус: MVP, end-to-end сценарий*
- Backend - 40 часов (API и основная логика)
- Frontend - 35 часов (экран выбора цветов)
- QA - 15 часов (тест-кейсы, smoke)
- PM - 20 часов (планирование, backlog, синки)

**Sprint 2** - доработка и качество (110 часов)
*Фокус: результат, UX, стабильность*
- Backend - 35 часов (оптимизация, логирование)
- Frontend - 35 часов (экран результата)
- QA - 20 часов (regression, UI тесты)
- PM - 20 часов (приёмка, демо, документация)

---

## Definition of Done

Задача считается выполненной, если:
- код в основной ветке
- пройдено code review
- нет критических багов
- тесты выполнены
- документация обновлена

---

## Риски

**Риск 1:** Непонятная интерпретация цветов
 *Митигирование:*
 - Простая и прозрачная логика
 - Проверка на фокус-группе

**Риск 2:** Перерасход часов
 *Митигирование:*
 - Оценки задач перед спринтом
 - Приоритет MVP

**Риск 3:** UX не подходит детям
 *Митигирование:*
 - Минимум текста
 - Визуальный фокус
